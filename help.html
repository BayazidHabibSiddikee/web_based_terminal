⚡ Quick Start

[INFO] AVR Control Plane compiles your .c file with avr-gcc and flashes it to the target chip using avrdude. Both tools must be installed.

1 — Install the AVR toolchain

PlatformCommand
Linux (Debian/Ubuntu)sudo apt install gcc-avr avr-libc avrdude
Linux (Fedora)sudo dnf install avr-gcc avr-libc avrdude
Linux (Arch)sudo pacman -S avr-gcc avr-libc avrdude
macOSbrew install avr-gcc avrdude
WindowsInstall WinAVR or scoop install avr-gcc avrdude
Termux (Android)pkg install avr-toolchain avrdude

2 — Start the backend

python3 terminal.py          # Linux / macOS / Termux
python  terminal.py          # Windows

The app opens automatically. If PySide6 is not installed it falls back to your system browser at http://localhost:8090.

3 — Write and flash your program

  1. 1
    Wire your programmer to the target chip (see Wiring section).
  2. 2
    Select TARGET_MCU matching your chip.
  3. 3
    Select PROGRAMMER — Arduino as ISP is the most common choice for external chips.
  4. 4
    Enter the FILE_PATH to your .c source, e.g. /home/user/blink (no extension).
  5. 5
    Select SERIAL_PORT (click SCAN to auto-detect) and press RUN_PROCESS.

🔌 Wiring Diagrams

[IMPORTANT] To flash an external chip (not the Arduino's own MCU), you must use Arduino as ISP mode. First upload the File → Examples → 11.ArduinoISP → ArduinoISP sketch to your Arduino, then come back here.

Arduino Uno as ISP → ATmega32A / ATmega32 / ATmega16 (DIP-40)

Arduino Uno (ISP mode) → ATmega32A / ATmega32 / ATmega16 (same ISP pinout). Add a 10µF capacitor between RESET and GND on the Arduino to prevent auto-reset.

Arduino Uno as ISP → ATmega328P (DIP-28) Standalone

ATmega328P standalone (no bootloader needed). Use Arduino as ISP programmer in the UI at 19200 baud.

ATmega328P key pins

Pin 1 — RESETPull-up 10kΩ to VCC recommended
Pin 7 — VCC3.3V or 5V
Pin 8 — GNDGround
Pin 17 — MOSI (PB3)← D11
Pin 18 — MISO (PB4)→ D12
Pin 19 — SCK (PB5)← D13
Pin 20 — AVCCConnect to VCC (ADC power)
Pin 22 — GNDGround (2nd pin)

Minimal power circuit

Place these components near the chip:

100nF capVCC → GND (bypass)
10µF capVCC → GND (bulk)
10kΩ resistorRESET → VCC (pull-up)
16 MHz crystalXTAL1 (pin9) + XTAL2 (pin10)
2× 22pF capEach XTAL pin → GND

Arduino Uno as ISP → ATtiny85 (DIP-8)

ATtiny85 DIP-8. ATtiny84 uses the same ISP pins but in a DIP-14 package — check the datasheet for pin numbers.
[ANTI-RESET CAP] Put a 10µF electrolytic capacitor between the Arduino's RESET pin and GND (positive leg to RESET). This prevents the Arduino from resetting itself when avrdude connects, which would break the ISP session.

📖 Procedures

Build & Flash

  1. 1
    Wire Arduino → target chip as shown in Wiring.
  2. 2
    Upload ArduinoISP sketch to the Arduino (do this once; it stays until you overwrite it).
  3. 3
    In the UI set TARGET_MCU, PROGRAMMER = Arduino as ISP, FILE_PATH (no .c), and SERIAL_PORT.
  4. 4
    Click RUN_PROCESS → BUILD & FLASH. Watch the terminal output for Flash Successful.

Generate .HEX only (no chip needed)

  1. 1
    Set TARGET_MCU and FILE_PATH.
  2. 2
    Change the action dropdown to GENERATE .HEX.
  3. 3
    Click RUN_PROCESS. The .hex file appears next to your source.

Serial Monitor

  1. 1
    Set SERIAL_PORT and BAUD RATE to match your firmware's UART_BAUD.
  2. 2
    Click START_MONITOR. Data from the chip appears in the terminal.
  3. 3
    The monitor auto-stops when you start a flash to free the port, and you must restart it manually after.

Writing fuses (clock source)

  1. 1
    Select CLOCK_SRC (e.g. Internal 8 MHz).
  2. 2
    Click WRITE_FUSES (orange button). This sets the correct lfuse for the selected clock.
  3. 3
    Warning: incorrect fuses can make the chip appear bricked. Use a slow BITCLOCK (32 or 128) with fresh chips.

🔒 Fuse Reference

[WARNING] Wrong fuse values can lock out your chip. Always double-check values before writing. If you brick a chip, you can recover it with a High-Voltage programmer.

avrdude -c stk500v1 -b 19200 -p m32a -P /dev/ttyACM0 \

⌨️ Useful Raw Terminal Commands

Type these in the RAW_TERMINAL box on the main screen.

Check tools are installed

avr-gcc --version
avrdude --version
avr-size --version

Verify chip connection (ATmega32A example)

avrdude -c stk500v1 -b 19200 -p m32a -P /dev/ttyACM0

Read fuses from chip

avrdude -c stk500v1 -b 19200 -p m32a -P /dev/ttyACM0 \
        -U lfuse:r:-:h -U hfuse:r:-:h

Flash a pre-built hex directly

avrdude -c stk500v1 -b 19200 -p m32a -P /dev/ttyACM0 \
        -U flash:w:/path/to/firmware.hex:i

Compile manually

avr-gcc -mmcu=atmega32a -Os -DF_CPU=8000000UL \
        -o blink.elf blink.c
avr-objcopy -O ihex -R .eeprom blink.elf blink.hex

Scan for serial ports (Linux)

ls /dev/ttyACM* /dev/ttyUSB*

Check chip memory usage

avr-size --format=avr --mcu=atmega32a blink.elf

List USB devices (Linux)

lsusb

🔧 Troubleshooting

avrdude: stk500_recv(): programmer not responding

  • Forgot to upload ArduinoISP sketch to the Arduino.
  • Wrong SERIAL_PORT selected — click SCAN to refresh.
  • Missing or loose wires — check all 6 connections.
  • No 10µF cap on Arduino RESET → GND (Arduino resets itself on connect).
  • Try a higher BITCLOCK value (32 or 128) for new chips on default 1 MHz.

Device signature mismatch / wrong chip

  • The UI will auto-detect the real chip and show [AUTO-DETECT] — change TARGET_MCU to match.
  • If using Arduino (Bootloader) programmer, it reads the Arduino's onboard 328P, not your external chip. Switch to Arduino as ISP.

avrdude: error: usbasp: could not find USB device

  • Linux: add udev rule — sudo nano /etc/udev/rules.d/99-usbasp.rules → add:
    SUBSYSTEMS=="usb", ATTRS{idVendor}=="16c0", MODE="0666"
  • Reload: sudo udevadm control --reload-rules

Cannot connect to backend

  • Run python3 terminal.py (Linux/Mac) or python terminal.py (Windows) in the project directory.
  • Make sure flash_avr.py is in the same folder as terminal.py.
  • Check port 8090 is not already in use: ss -tlnp | grep 8090

Chip looks bricked after wrong fuses

  • Use BITCLOCK 128 and try connecting again — the chip is usually still alive, just slow.
  • If it used an external crystal clock and you don't have a crystal connected, it will not respond. Connect a crystal to XTAL1/XTAL2 and try again.
  • Last resort: High-Voltage Serial Programmer (HVSP) or Parallel Programmer.

Compilation error: avr-gcc not found

  • Install the toolchain (see Quick Start).
  • On Windows, ensure WinAVR bin/ folder is in your PATH.
  • Test in RAW_TERMINAL: avr-gcc --version
[QUICK CHECK]   Green LED on the Arduino D9 blinks slowly → ISP sketch is running correctly and ready.
